# Architecture (for contributors)

Internals documentation, written for people who want to contribute to Trackdown itself, not just use it.  individual pages here go deeper on:

- Envelope pipeline (parsing, validation, dispatch)
- Grouping / fingerprinting algorithm
- Storage layout (SQLite schema, indexes, retention)
- Symbolication (sourcemap resolution)

Pages are added as each subsystem is built, not written speculatively ahead of the code.
